SidSearch-LoRA 0.5B
Experimental release

I froze 99.78% of a language model—and it still learned the task.

Only 0.2184% of Qwen2.5-0.5B-Instruct was trained. On a private held-out SidSearch benchmark, the LoRA-adapted model delivered a 20.1% relative composite gain.

99.78% of model parameters frozen
0.22% of model parameters trained
+20.1% relative composite improvement

The task

SidSearch converts natural-language search requests into JSON execution plans across documents, email, GitHub, and multi-source search. The model had to learn a private protocol it had not seen during pretraining.

The experiment

The adapter trained on 150 examples, used 25 examples for validation, and was evaluated against the untouched base model on the same 50 held-out examples.

Metric Base LoRA Change
Composite score 0.2542 0.3053 +20.1%
Intent accuracy 0.00 0.16 +0.16
Source accuracy 0.00 0.22 +0.22
Entity F1 0.06 0.24 +0.18

What changed

The LoRA adapter improved the model's understanding of task intent, source routing, and entity extraction while leaving the original base weights untouched.

A small model does not need every parameter retrained to become more capable at one clearly defined job.

Why it matters

LoRA makes specialization practical: train a compact adapter, preserve the base model, and measure the result on a benchmark the model never saw during training.

This is a narrow experimental result, not a claim of general model superiority. The full repository includes dataset generation, sequence-level distillation, frozen-weight verification, training code, and held-out evaluation.
