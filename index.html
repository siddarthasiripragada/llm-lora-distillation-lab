LoRA fine-tuning · sequence-level distillation · held-out evaluation

Teaching a 0.5B model a private search protocol

SidSearch is a small, reproducible experiment that fine-tunes only LoRA adapter weights while keeping the base model frozen. The benchmark asks whether the adapted model improves on an unseen structured-generation task.

Result

Base closed-book composite0.2542
LoRA closed-book composite0.3053
Trainable parameter fraction0.2184%

The LoRA adapter improved the held-out composite score by 0.0511 absolute. The result is intentionally reported with its limitations: exact full-record match and schema compliance remain at zero in this run.

Task

SidSearch converts an internal search request into a strict JSON execution plan. It tests whether a model can learn a private schema, source-routing rules, date rules, ambiguity handling, and filter extraction. 

Input:
Search unread email from Morgan about InvoicePilot

Output:
{
  "intent": "email_search",
  "entities": ["InvoicePilot"],
  "source": "email",
  "filters": {
    "start_date": null,
    "end_date": null,
    "owner": "Morgan",
    "file_type": null,
    "status": "unread"
  },
  "confidence": "high",
  "clarification_required": false
}

Method

1. ProtocolDefine SidSearch rules and schema.
2. ScenariosGenerate private task examples.
3. TeacherUse Ollama qwen2.5:3b for sequence labels.
4. ValidateReject malformed or rule-breaking outputs.
5. TrainFreeze base model, update LoRA only.
6. BenchmarkCompare on 50 held-out examples.

The training data comes from validated sequence-level distillation. The teacher produces full JSON target sequences; the student is trained with supervised fine-tuning. This is not logit distillation.

Claim Boundary

Supported claim: on 50 held-out SidSearch examples, the LoRA-adapted 0.5B model improved composite task score from 0.2542 to 0.3053 while training only 0.2184% of parameters.

  • No claim of general model improvement.
  • No claim of production readiness.
  • No claim that training-set performance demonstrates generalization.
  • No claim outside the SidSearch benchmark.
